# Web Platform Tests

This directory contains a pinned subset of the upstream
[Web Platform Tests](https://github.com/web-platform-tests/wpt/tree/master/url)
for the WHATWG URL Standard.

## Generated snapshot

Files listed in `checksums.json` are generated by `yarn update-wpt` from the
pinned upstream revision. Do not edit them manually. Make changes upstream or
update `wpt.lock.json`, then regenerate the snapshot. Files such as this README,
`wpt.lock.json`, `exceptions.json`, and `hermes.json` are maintained in this
repository.

## Running

```bash
yarn test:wpt
```

`OfficialWPT-test.js` runs one Jest test for each upstream WPT file and declared
variant. `scripts/wpt/run-wpt.js` evaluates the package implementation and the
official WPT harness in an isolated JavaScript realm. It supports upstream
`META` scripts, asynchronous tests, local `fetch()` resources, and
`idlharness.any.js`.

The runner uses the official pinned copies of:

- `testharness.js`
- `WebIDLParser.js`
- `idlharness.js`
- `subset-tests-by-key.js`
- `interfaces/url.idl`

The resource-driven constructor, origin, and setter files execute unchanged.
They load their checked-in JSON through the runner's local resource mapping.

## Pinning

`wpt.lock.json` records the exact upstream commit, test inventory, and excluded
files. `checksums.json` records every downloaded test, resource, and harness
dependency.

To synchronize the pinned snapshot:

```bash
yarn update-wpt
```

The update command downloads every file from the lock's commit. It also compares
the lock against the complete upstream `url/*.any.js` inventory and fails if a
file has not been included or explicitly excluded.

To update WPT, change the revision in `wpt.lock.json`, run `yarn update-wpt`,
inspect every changed test and harness dependency, then run `yarn test:wpt`.

## Hermes and React Native

The React Native `0.84` Android Detox fixture runs the pinned WPT files and
variants that exercise this URL polyfill in Hermes. Generate its Metro input
with:

```bash
yarn generate:wpt:react-native
```

The generated `platforms/detox/generated/hermes-wpt.js` file embeds the pinned
tests, harness, resources, and exception titles. It is ignored by Git and must
not be edited or committed. The React Native fixture build regenerates it after
verifying `checksums.json`.

`hermes.json` records React Native host-environment differences separately from
polyfill exceptions. The Hermes lane excludes `urlencoded-parser.any.js`
because it tests `Request.formData()` and `Response.formData()`, and allows the
single `FormData` constructor case because React Native's `FormData` is not
iterable.

Detox launches each case through a `wpt://run` deep link with
`newInstance: true`. This gives each WPT file or variant a fresh app process and
Hermes realm.
The app reports unexpected failures and stale exceptions through
`hermes-wpt-result`; `platforms/detox/hermesWpt.test.js` requires every case to
pass. The fixture also verifies `global.HermesInternal`, so this lane cannot
silently run on JavaScriptCore.

## Exceptions

`exceptions.json` is an exact title-based allowlist. The runner fails for both
unexpected failures and stale exceptions that have started passing.

The current exceptions are URL cases requiring IDNA processing, which this
package intentionally omits for bundle size. The full upstream IDNA suites are
classified separately in `wpt.lock.json` for the same reason.

Fetch `Request.formData()` and `Response.formData()` cases execute using the
host runtime. They validate harness completeness but do not exercise this URL
implementation directly.
